AgentConvert v2

私有 AI 中继平台，支持池组、独立计费倍率、用户邀请机制和可审计的配额管理。

TypeScript Node.js 22+ npm 10+ SQLite

概述

AgentConvert v2 是一个私有 AI 中继平台，通过可配置的池组路由模型请求，每个池组拥有独立的计费倍率。平台提供完整的用户生命周期管理，包括基于邀请的注册、配额管理，以及所有财务操作的完整审计追踪。

平台支持多个上游提供商（OpenRouter、OpenAI PPAP、Claude PPAP），每个池组可设置不同的费用倍率，实现对组织内计费和资源分配的精细控制。

核心功能

  • 池组支持独立计费倍率（OpenRouter 10x、OpenAI PPAP 1x、Claude PPAP 2x）
  • 用户邀请码支持充值触发的 token 奖励（邀请人 5% 返利）
  • 配额、token 和充值变更全程审计，完整可追溯
  • 管理员池组维护和审计端点
  • 安全基线强制 HTTPS 来源和 session 密钥
  • 本地端到端测试基线，保障开发信心
  • 数据一致性保证：防御性拷贝、重放幂等性、溢出拒绝

技术栈

基于现代 TypeScript 构建，专注于类型安全、可审计性和最小运维开销。SQLite 提供零配置持久化，适合平台的单节点部署模型。

  • 运行时：Node.js 22+，原生 TypeScript 支持
  • 语言：TypeScript strict 模式
  • 包管理器：npm 10+
  • 数据库：SQLite（嵌入式，零配置）
  • 测试：本地端到端测试套件

使用指南

平台提供一组内部和面向用户的 API 端点，用于模型调用计费、充值处理、邀请管理和管理员操作。

API 端点

管理员端点

http 
GET  /api/admin/pool-groups          # List pool groups
PATCH /api/admin/pool-groups/:slug   # Update pool group
GET  /api/admin/audit                # Audit log

内部端点

http 
POST /api/internal/model-call-charges # Record model-call charge
POST /api/internal/recharge-events   # Record recharge event

用户端点

http 
POST  /api/user/invitation-codes      # Create invitation
GET   /api/user/invitation-codes      # List user codes
PATCH /api/user/invitation-codes/:id  # Disable code
POST  /api/auth/register              # Register with invitation
GET   /app/invitations                # Invitation desk page

健康检查

服务在默认地址暴露健康检查端点，用于监控和就绪探测。

bash 
curl http://127.0.0.1:3217/health

开发指南

项目使用简洁的基于 npm 的工作流。所有检查、测试和开发服务器均可通过标准脚本访问。

前置要求

  • Node.js 22+
  • npm 10+

本地配置

bash 
# Install dependencies
npm install

# Start development server (http://127.0.0.1:3217)
npm run dev

# Run type checks and linting
npm run check

# Run local end-to-end tests
npm run e2e:local

Git 工作流

仓库采用分层分支策略，确保从开发到生产的安全推进。

text 
master          # Production
  release       # Release preparation
    dev         # Integration branch
      feature/* # Feature branches
  • feature/*：所有新工作从此开始，基于 dev 分支创建
  • dev：集成分支，功能在此合并并联合测试
  • release：预生产暂存，用于最终验证
  • master：仅包含生产就绪代码
