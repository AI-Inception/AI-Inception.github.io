Video Digest

AI-powered video summarizer Telegram bot. Paste a YouTube or Bilibili URL, get a structured summary with timestamped key points.

Python 3.12 FastAPI Claude Sonnet 5 yt-dlp faster-whisper SQLite

Overview

Video Digest is an AI-powered pipeline that transforms long-form video content into concise, structured summaries. Users interact through a Telegram bot interface or WebUI — paste a YouTube or Bilibili URL, and the system extracts transcripts, processes them through an LLM, and returns a summary with timestamped key points.

The system is designed to handle videos of any length. Transcripts exceeding 30K tokens are automatically chunked into approximately 8K segments with overlap, summarized individually, then merged into a coherent final output. Results are cached in SQLite with a 7-day TTL to avoid redundant processing.

Key Features

  • Intelligent chunking for long transcripts (approximately 8K per chunk with overlap), with per-chunk summarization followed by a merge pass
  • Dual transcript extraction: yt-dlp subtitle extraction as primary method, with audio-to-text via faster-whisper as fallback
  • SQLite result caching with 7-day TTL eliminates redundant API calls for repeated URLs
  • Server-Sent Events (SSE) for real-time progress streaming to clients during processing
  • LLM summarization powered by Claude Sonnet 5 via AgentConvert gateway

Tech Stack

Built on a Python async stack with FastAPI for high-concurrency request handling and streaming responses.

  • Runtime: Python 3.12, FastAPI, uvicorn
  • Transcript Extraction: yt-dlp (subtitle download), faster-whisper (speech-to-text fallback)
  • LLM: Claude Sonnet 5 via AgentConvert gateway
  • Storage: SQLite (result cache, 7-day TTL)
  • Streaming: Server-Sent Events (SSE) for real-time progress
  • Package Management: uv

Usage

The processing pipeline follows a deterministic flow from URL input to structured summary output.

Pipeline

text 
URL -> yt-dlp subtitle extraction -> (fallback: audio -> faster-whisper) -> LLM summary -> WebUI

Chunking Strategy

For transcripts exceeding 30K tokens, the system splits content into chunks of approximately 8K tokens with overlap between adjacent chunks to preserve context continuity. Each chunk is summarized independently, then the per-chunk summaries are merged into a single coherent output in a final LLM pass.

Development

Prerequisites

  • Python 3.12+
  • uv (package manager)
  • API keys configured in .env

Local Setup

bash 
cp .env.example .env   # fill in API keys
uv sync
uv run python -m video_digest.main

Deployment

Production deployment runs on a Raspberry Pi 5 as a systemd service on port 8899.
